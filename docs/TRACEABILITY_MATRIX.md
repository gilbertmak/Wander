# FP-9.3 Traceability Matrix

Source of truth: `docs/traceability.json`

## Release Gates

- Every requirement row has an epic, data rule, acceptance criteria ID, automated test ID, manual evidence, UI surface, release gate, and owner.
- Automated test IDs must point to files that exist in the repository.
- Security and database gates require additional command evidence before release: temporary database migration, log redaction tests, export tests, and source-file exclusion checks.
